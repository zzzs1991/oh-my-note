#### 1. 如何学习Linux性能优化
##### 路径
1. 理解应用程序和系统的少数几个基本原理
2. 大量实战练习
3. 建立整体性能的全局观
##### 性能指标是什么
高并发、响应快 --> 吞吐、延时 --> 应用负载的视角 --> 系统资源的视角 --> 资源使用率、饱和度

1. 应用负载
2. 系统资源

1. Applications
2. Libraries
3. System Call
4. Linux Kernel
5. Device

`性能问题的本质: 系统资源达到瓶颈`
找出和避免、缓解它们，从而更高效的利用系统资源去处理更多的请求。

###### 性能分析的六个步骤
1. 选择指标评估应用程序和系统的性能
2. 为应用程序和系统设置性能目标
3. 进行系统基准测试
4. 性能分析定位瓶颈
5. 优化系统和应用程序
6. 性能监控和告警

###### 技巧
1. 刚开始不要试图抓住所有实现细节
2. 边学边实践
3. 勤思考，多反思，善总结
